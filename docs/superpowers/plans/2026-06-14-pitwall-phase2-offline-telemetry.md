# PitWall — Phase 2 (offline) — Bundled Telemetry — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development (or executing-plans).
> Steps use checkbox (`- [ ]`) syntax. **This plan SUPERSEDES the backend plan**
> (`2026-06-14-pitwall-phase2-telemetry-backend.md`) for execution — Hartej chose the **offline bundle**
> path (no server, no Oracle, no ops). That doc is retained only as the record of the road not taken.

**Goal:** Add per-lap car telemetry (speed/throttle/brake/gear/DRS/X-Y), delta-time, track-dominance,
tyre degradation, and a race-pace enrichment of the Driver-vs-Car hero — entirely **offline**, served from a
second bundled read-only SQLite (`telemetry.db`) produced by a **local** FastF1 bake script, distributed like
`f1db.db`. Zero backend, $0, no account, no network on any screen.

**Architecture:** A Python **bake** (`tools/bake/`, run locally with FastF1) loads each 2018+ session in ONE
`session.load(laps=True, telemetry=True, ...)` call, decimates telemetry **by distance**, and writes compact
**parallel arrays (comma-delimited strings)** + race-pace/tyre-deg rows into `telemetry.db`. The app opens that
file through a **second SQLDelight database** (`TelemetryDb`), mirroring the existing f1db open-path
(composeResources → app dir → open, never `Schema.create`). All telemetry domain logic is **pure + TDD**;
charts are **Compose Canvas** (no chart lib — matches the app's no-nav-lib/no-icon-pack minimalism).

**Tech Stack:** Existing (Kotlin 2.2.20 / Compose MP 1.11.1 / SQLDelight 2.3.2 / Koin 4.2.1, JDK 17) **plus**
a second SQLDelight DB. **No new Kotlin runtime deps** (channels are delimited strings → stdlib parse; gzip
avoided in v1). Bake side: Python ≥3.10 + `fastf1==3.8.3`, `pandas<3`, `numpy<3`, `scipy<2`; tests use
stdlib `unittest` (pure transforms take plain lists, so no pandas/fastf1 needed to test them).

## Hard constraints carried forward (these override convenience)

| Constraint | How this plan honors it |
|---|---|
| **$0 / no ops** | No server at all. Local bake → static `telemetry.db`. Distribution = the file ships in the app bundle (and, later, an optional GitHub-Releases download — free static hosting, no API). |
| **Offline-first** | The app reads only the bundled `telemetry.db` + `f1db.db`. No network on any screen in this phase. |
| **No "F1"/marks branding** | Table names, file names, UI strings use "PitWall"/neutral terms. Telemetry is described as "car data", "2018+". |
| **CC-BY attribution** | LicensesScreen + `DATASET_LABEL` extended to credit **FastF1** (+ its upstream sources) alongside the existing f1db CC-BY line. |
| **Personal PUBLIC repo** | Bake code is public OSS. The real `telemetry.db` is **gitignored** (like `f1db.db`); a synthetic dev seed is generated by a committed script, never a real-data blob in git history. |

## Net-deltas from the backend plan (what changed by going offline)

1. **Section A (FastAPI/Postgres/Redis/Oracle/Caddy/keep-alive/auth) is DELETED.** Replaced by `tools/bake/`
   writing a SQLite file. ~90% less to build, 100% less to run.
2. **No Ktor / no persistent HTTP cache / no API key** (was Section B Task 2). Replaced by a second SQLDelight
   DB opened exactly like f1db. (Optional Releases-download updater is a *named future increment*, not v1.)
3. **Blobs are not gzipped and not array-of-objects.** Channels are **comma-delimited `TEXT` per channel**
   (`"0.0,10.5,21.2"`), parsed by a pure stdlib helper on-device. Zero serialization deps. (Gzip is a future
   optimization if bundle size demands it.)
4. **Charts are Canvas, not Vico.** No chart-lib dependency; consistent with the existing house style.
5. **Freshness model:** data is as-of the last bake shipped in the bundle (a `telemetry.version` stamp +
   `TELEMETRY_LABEL`, mirroring `DATASET_VERSION`/`DATASET_LABEL`). Re-bundle to refresh. Carried correctness
   facts from the backend plan that STILL apply: single `session.load(...)` call (#852 dup bug); telemetry is
   2018+ only (pre-2018 → graceful "results only"); decimate **by distance** (keeps cross-driver overlays aligned).

## The contract: `telemetry.db` schema (keystone — lock first)

```sql
-- tools/bake/schema.sql  (and 1:1 the CREATE TABLEs in Telemetry.sq)
CREATE TABLE session (
  id            TEXT PRIMARY KEY,   -- "2024-1-R"  (year-round-sessionType)
  year          INTEGER NOT NULL,
  round         INTEGER NOT NULL,
  session_type  TEXT NOT NULL,      -- 'R','Q','S','SQ','FP1'...
  event_name    TEXT NOT NULL,      -- neutral GP name, e.g. "Bahrain"
  circuit_name  TEXT NOT NULL,
  baked_at      TEXT NOT NULL       -- ISO date the bake ran
);
CREATE TABLE lap (
  session_id    TEXT NOT NULL,
  driver_id     TEXT NOT NULL,      -- f1db driver slug where resolvable, else FastF1 abbreviation
  driver_label  TEXT NOT NULL,      -- display "Max Verstappen (VER)"
  lap_number    INTEGER NOT NULL,
  lap_time_ms   INTEGER,            -- nullable (in/out/no-time laps)
  compound      TEXT,               -- SOFT/MEDIUM/HARD/INTER/WET
  tyre_life     INTEGER,
  stint         INTEGER,
  is_accurate   INTEGER NOT NULL DEFAULT 0,  -- 0/1
  PRIMARY KEY (session_id, driver_id, lap_number)
);
CREATE TABLE telemetry_channel (   -- one row per driver-lap; channels are comma-delimited strings
  session_id    TEXT NOT NULL,
  driver_id     TEXT NOT NULL,
  lap_number    INTEGER NOT NULL,
  distance      TEXT NOT NULL,     -- "0.0,10.0,20.0,..."  (decimated by distance)
  speed         TEXT, throttle TEXT, brake TEXT, gear TEXT, drs TEXT, x TEXT, y TEXT,
  PRIMARY KEY (session_id, driver_id, lap_number)
);
CREATE TABLE driver_pace (         -- race-pace companion + tyre-deg fit per driver-session
  session_id        TEXT NOT NULL,
  driver_id         TEXT NOT NULL,
  driver_label      TEXT NOT NULL,
  median_pace_ms    INTEGER,       -- median accurate green-flag lap
  pace_pct_vs_team  REAL,          -- symmetric % vs teammate (negative = faster); NULL if no teammate
  deg_ms_per_lap    REAL,          -- fitted linear tyre-deg slope (ms/lap), fuel-corrected
  PRIMARY KEY (session_id, driver_id)
);
```

> **SQLDelight 2.x multi-database gotcha:** both databases default to reading `src/<set>/sqldelight`. Give the
> new DB a **separate `srcDirs`** (`src/commonMain/sqldelight-telemetry`) and a distinct `packageName`
> (`dev.pitwall.telemetrydb`) so the two never compile each other's `.sq`. Verify by a clean
> `:composeApp:generateCommonMainTelemetryDbInterface` + compile. (One `*Queries` class per `.sq` file →
> accessor is `telemetryDb.telemetryQueries`.)

```python
# tools/bake/tests/test_transform.py
import unittest
from pitwall_bake.transform import decimate_by_distance, pack_channels, symmetric_pace_pct, tyre_deg_slope

class TransformTest(unittest.TestCase):
    def test_decimate_keeps_endpoints_and_spacing(self):
        dist  = [i * 1.0 for i in range(0, 101)]          # 0..100 m, 1 m apart
        speed = [100.0 + i for i in range(0, 101)]
        d2, chans = decimate_by_distance(dist, {"speed": speed}, step_m=25.0)
        self.assertEqual(d2[0], 0.0)
        self.assertEqual(d2[-1], 100.0)                    # last sample always kept
        self.assertTrue(all(b - a >= 25.0 - 1e-9 for a, b in zip(d2, d2[1:])))
        self.assertEqual(len(d2), len(chans["speed"]))

    def test_pack_channels_csv_roundtrip(self):
        self.assertEqual(pack_channels([0.0, 10.5, 21.25]), "0.0,10.5,21.25")
        self.assertIsNone(pack_channels(None))

    def test_symmetric_pace_pct_sign(self):
        # driver 90.0s vs teammate 90.9s -> driver faster -> negative
        self.assertLess(symmetric_pace_pct(90000, 90900), 0.0)
        self.assertAlmostEqual(symmetric_pace_pct(90000, 90000), 0.0, places=9)

    def test_tyre_deg_slope_positive_when_degrading(self):
        # lap times rising with tyre life -> positive ms/lap slope
        self.assertGreater(tyre_deg_slope([1, 2, 3, 4], [90000, 90200, 90400, 90600]), 0.0)
```

Expected: FAIL (module missing).

- [ ] **Step 2: implement `transform.py`** (pure, stdlib only):
  - `decimate_by_distance(distance, channels: dict, step_m)`: walk the distance axis keeping a sample whenever
    cumulative distance since the last kept sample ≥ `step_m`; **always keep the final sample**; subset every
    channel to the kept indices; return `(kept_distance, {name: kept_values})`.
  - `pack_channels(values)`: `None`→`None`; else `",".join(repr-free str(v))` (use `("%g"%v)` for floats, plain
    for ints) — compact, lossless-enough for decimated telemetry.
  - `symmetric_pace_pct(ms_i, ms_j)`: `100*(ms_i-ms_j)/((ms_i+ms_j)/2)` (mirror of the hero's `symmetricGapPct`).
  - `tyre_deg_slope(tyre_life, lap_ms)`: least-squares slope of `lap_ms` vs `tyre_life` (stdlib math; guard <2 pts → 0.0).

- [ ] **Step 3: run → PASS.**
